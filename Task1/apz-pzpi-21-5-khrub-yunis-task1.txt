1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1 Background (Передумови)

У сучасному медичному світі існує нагальна потреба у зручних та ефективних інструментах для доступу до інформації про лікарські засоби, обміну знаннями та досвідом між лікарями та фармацевтами, а також у зміцненні їхніх професійних мереж. Незважаючи на значний обсяг інформації, що існує в медичній галузі, доступ до неї може бути обмеженим та незручним для медичних працівників. У сучасному медичному ландшафті, де швидкість та точність рішень мають критичне значення, інструмент, який може забезпечити швидкий та надійний доступ до інформації про лікарські засоби, може стати ключовим для покращення якості надання медичних послуг.Ця платформа відповідає на зростаючу потребу в інструменті, який об'єднує медичних фахівців та фармацевтів для спільної роботи, обміну знаннями та досвідом. Вона допомагає у зборі, організації та представленні інформації про лікарські засоби у зручний та доступний спосіб, забезпечуючи медичним працівникам засіб для швидкого прийняття інформованих рішень щодо лікування пацієнтів.

1.2 Business Opportunities (Бізнес-можливості)

Існує великий потенціал для платформи, призначеної для медичних працівників, що може допомогти у полегшенні пошуку інформації про лікарські засоби, обміну знаннями та досвідом між лікарями та фармацевтами, а також у розвитку їхніх професійних мереж. Для нашого продукту відкритий широкий ринок медичних працівників, які постійно шукають ефективні інструменти для підтримки своєї практики та розвитку своїх професійних навичок. Основні можливості, що надає наша платформа, включають:
•	Полегшення доступу до інформації про лікарські засоби: Наша платформа забезпечить медичним працівникам швидкий та зручний доступ до найактуальнішої інформації про лікарські засоби, включаючи описи, дозування, побічні ефекти та інші важливі дані.
•	Обмін знаннями та досвідом: Ми надамо можливість лікарям та фармацевтам обмінюватися своїм досвідом у використанні лікарських засобів, розглядати клінічні випадки та найкращі практики, що допоможе підвищити якість медичної практики.
•	Розвиток професійних мереж: Наша платформа створить унікальну спільноту медичних працівників, що сприятиме розвитку їхніх професійних мереж, сприяє колаборації та спільному вирішенню проблем.
Щодо можливостей монетизації, наша компанія може реалізувати такі стратегії:
•	Підписка на платформу для користувачів: Ми можемо встановити щомісячну або щорічну плату за доступ до всіх функцій та ресурсів платформи.
•	Рекламні можливості: Ми можемо надати можливість рекламодавцям цільово розміщувати рекламу на платформі, спрямовану на медичних фахівців.
•	Партнерські програми: Ми можемо співпрацювати з фармацевтичними компаніями та іншими стейкхолдерами для створення спеціальних пропозицій та програм для користувачів платформи.
Загальна потенційна прибутковість нашого проєкту полягає в тому, що ми вирішуємо актуальну проблему медичної галузі та надаємо цінний інструмент для підтримки професійного зростання медичних працівників.

2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1 Vision Statment (Окреслення концепції)

Наша мета - створити інноваційну платформу, яка революціонізує спосіб, яким медичні працівники отримують і обмінюються інформацією про лікарські засоби, набуті знання та професійні досвіди. Наша платформа буде відомим місцем для медичної спільноти, де вони зможуть легко знаходити потрібну інформацію, спілкуватися з колегами, обмінюватися найкращими практиками та розвивати свої професійні мережі.Наша платформа для медичних працівників буде високоефективним інструментом, що об'єднує усі аспекти їхньої професійної діяльності. Вона буде простою у використанні, інтуїтивно зрозумілою та надійною. Ми прагнемо створити середовище, де медичні фахівці зможуть легко знаходити необхідну інформацію про лікарські засоби, ділитися своїми знаннями та досвідом, а також розширювати свої професійні зв'язки. Наша платформа покликана полегшити робочий процес медичних працівників та сприяти підвищенню якості медичної практики.

2.2 Major Features (Головна функціональність)

MF-1: Повний доступ та керування користувачами: Можливість реєстрації та керування користувачами, що дозволяє легко впроваджувати та керувати доступом до платформи для медичних працівників.
MF-2: Пошук та обмін інформацією: Функціонал, який забезпечує легкий пошук інформації про лікарські засоби, а також можливість обміну знаннями та досвідом між лікарями та фармацевтами.
MF-3: Аналітика та звітність: Можливість отримання статистики та аналітики щодо використання платформи, що допоможе вдосконалити процеси та приймати обгрунтовані рішення.
MF-4: Забезпечення безпеки даних: Автоматичне хешування паролів та зберігання їх у хешованому вигляді у базі даних для забезпечення високого рівня безпеки даних користувачів.
MF-5: Розвиток професійних мереж: Функціонал, що дозволяє медичним працівникам розвивати свої професійні мережі, знаходити колег та експертів для консультацій та співпраці.

3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ)

3.1 Scope of Initial Release (Рамки первинного випуску)

Програмне забезпечення серверної частини системи:
Для серверної частини системи буде розроблено програмне забезпечення, яке забезпечить обробку та зберігання даних від IoT пристроїв, управління користувачами та інтеграцію з веб та мобільними застосунками. Основні функції включатимуть:
•	Обробку запитів від IoT пристроїв для зберігання та аналізу даних.
•	Управління користувачами, включаючи реєстрацію, аутентифікацію та авторизацію.
•	Підтримку RESTful API для взаємодії з веб та мобільними застосунками.
•	Моніторинг та логування системних подій для забезпечення надійності та безпеки системи.
•	Інтеграцію з базою даних для зберігання інформації про користувачів, пристрої та зібрані дані.
Програмне забезпечення пристроя IoT:
Програмне забезпечення для IoT пристроїв буде розроблено для збору та передачі даних до серверної частини системи. Основні функції включатимуть:
•	Збір даних з датчиків пристрою в реальному часі.
•	Передача зібраних даних на сервер за допомогою мережевих протоколів (наприклад, HTTP, MQTT).
•	Підтримку механізмів безпеки для захисту переданих даних.
•	Можливість віддаленого оновлення програмного забезпечення пристрою.
•	Інтерфейс для локальної конфігурації та діагностики пристрою.
Веб застосунок:
Веб застосунок буде розроблено для надання користувачам доступу до даних та функцій системи через веб-інтерфейс. Основні функції включатимуть:
•	Інтерфейс для реєстрації та аутентифікації користувачів.
•	Панель керування для перегляду та аналізу даних, зібраних з IoT пристроїв.
•	Інтерфейс для керування користувачами та налаштуваннями системи.
•	Відображення даних у вигляді графіків та таблиць для полегшення аналізу.
•	Підтримку адаптивного дизайну для зручного використання на різних пристроях (десктоп, планшет, смартфон).
Програмний застосунок для мобільного пристроя:
Мобільний застосунок буде розроблено для забезпечення користувачам доступу до даних та функцій системи на мобільних пристроях. Основні функції включатимуть:
•	Інтерфейс для реєстрації та аутентифікації користувачів.
•	Панель керування для перегляду та аналізу даних, зібраних з IoT пристроїв.
•	Повідомлення та сповіщення про важливі події та оновлення даних.
•	Інтерфейс для керування налаштуваннями системи та пристроїв.
•	Підтримку офлайн-режиму для доступу до попередньо завантажених даних без підключення до інтернету.
•	Інтеграцію з мобільними сенсорами для додаткового збору даних (якщо застосовно).
Ці компоненти разом забезпечать повнофункціональну систему для збору, обробки та аналізу даних з IoT пристроїв, надаючи користувачам інтуїтивно зрозумілий інтерфейс для взаємодії з системою через веб та мобільні застосунки.

3.2 Scope of Subsequent Releases (Рамки наступних випусків) 

Підписки на подальші випуски платформи для медичних працівників будуть спрямовані на розширення та поліпшення існуючих функціональностей, а також впровадження нових можливостей для подальшого покращення користувацького досвіду та відповідності змінюваним потребам. Обсяг подальших релізів може включати:
1.	Розширене керування профілями користувачів:
•	Розширені опції налаштувань для профілів користувачів, які дозволять медичним працівникам вказати свої спеціалізації, сертифікати, наукові інтереси та афіліації.
•	Інтеграція з професійними мережами для забезпечення зручного оновлення профілів та можливостей мережування.
2.	Розширене керування знаннями:
•	Впровадження розширених алгоритмів пошуку та фільтрів для надання більш точних і актуальних результатів пошуку.
•	Інтеграція з зовнішніми медичними базами даних або ресурсами для розширення доступної інформації та ресурсів.
•	Впровадження розділів з кураторським контентом, рекомендацій експертів та персоналізованих шляхів навчання на основі вподобань користувачів та їх активності.
3.	Інструменти співпраці та комунікації:
•	Впровадження розширених функцій співпраці, таких як групові обговорення, інструменти управління проектами та спільне редагування документів.
•	Інтеграція з платформами телемедицини або інструментами відеоконференцій для забезпечення віртуальних консультацій та міждисциплінарної співпраці.
4.	Професійний розвиток та навчання:
•	Інтеграція ресурсів неперервної освіти, акредитованих курсів та програм сертифікації в межах платформи.
•	Розробка інтерактивних навчальних модулів, вивчення клінічних випадків та симуляцій для підтримки постійного професійного розвитку та підвищення навичок.
5.	Аналітика та звіти:
•	Впровадження розширених інструментів аналітики для отримання практичних результатів з активності користувачів, взаємодії з контентом та спільної діяльності.
•	Надання панелей приладів та звітів для відстеження ключових показників ефективності, метрик залучення користувачів та тенденцій використання платформи.
6.	Поліпшення мобільних додатків:
•	Подальша оптимізація мобільних додатків для поліпшення продуктивності, зручності використання та функціональних можливостей порівняно з настільними версіями.
•	Впровадження можливостей офлайн-доступу, що дозволить користувачам отримувати доступ до основних функцій та контенту навіть без підключення до Інтернету.


4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)

4.3 Operating Environment (Робоче середовище)

Система повинна бути доступною 24/7, оскільки користувачі можуть потребувати використання системи у будь-який час. Це вивимагає високого рівня доступності серверів та мережевих ресурсів. Надійність системи важлива для запобігання пере та для забезпечення безпеки користувачів. Засоби резервування та відновлення повинні бути встановлені для зменшення можливих відмов.Система повинна забезпечувати високу продуктивність для обробки даних. Це може включати оптимізацію бази даних та мережевого обладнання Дані повинні бути захищені від несанкціонованого доступу та змін. Це вимагає використання сучасних методів шифрування та засобів безпеки 
Для використання ПК версією програми знадобиться ПК з ОС Windows, для мобільної версії – Android.
Для серверної частини буде використано мову програмування C# та через графічний інтерфейс буде можливо виконувати функції, які пов'язані з сервером
Для мобільного застосунку буде використано мову програмування Kotlin.
Для розробки Смарт-девайсу буде використовуватися Arduino IDE.
У якості СУБД для структурованих даних буде використовуватися СУБД MS SQL Server.
